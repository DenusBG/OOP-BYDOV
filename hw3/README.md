# Доповідь: Принципи ISP та DIP

## 1. Принцип розділення інтерфейсу (ISP)
Клієнти не повинні залежати від методів, які вони не використовують.

**Приклад порушення:** Інтерфейс `ISmartDevice` з методами `Print()`, `Scan()`, `Fax()`. Якщо звичайний принтер не вміє сканувати, він все одно змушений реалізувати порожній метод `Scan()`.
**Рішення:** Розділити на `IPrinter`, `IScanner` та `IFaxer`.

## 2. Принцип інверсії залежностей (DIP)
Модулі високого рівня не повинні залежати від модулів низького рівня. Обидва повинні залежати від абстракцій.

**Переваги DI:**
* Можливість замінити реалізацію (наприклад, базу даних) без зміни логіки.
* Легке тестування (використання Mock-об'єктів).

## 3. Взаємозв'язок ISP та DI
"Вузькі" інтерфейси роблять впровадження залежностей точнішим: клас отримує лише ті інструменти, які йому справді потрібні для роботи.
